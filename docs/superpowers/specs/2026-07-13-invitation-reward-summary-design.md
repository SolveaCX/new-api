# 邀请页配置化奖励说明设计

- 日期：2026-07-13
- 分支：`feature/invitation-tab`
- 状态：已确认，待实施

## 目标

在邀请页“邀请好友，赚取奖励”标题正下方增加一段奖励说明。说明必须使用当前邀请奖励配置，不能硬编码 `$20` 或“无上限”。

## 展示规则

说明使用 `GET /api/user/self/invitations` 返回的摘要字段：

- `inviter_reward_usd`：邀请者获得的 API 额度。
- `invitee_reward_usd`：受邀者获得的 API 额度。
- `inviter_reward_max_count`：邀请者可获得奖励的次数上限；`0` 表示不限制次数。

金额相同时显示“双方各得”，例如：

> 邀请好友注册并完成首次充值后，双方各得 $20 API 额度。无上限、不过期，任意邮箱均可。

金额不同时分别说明双方所得，例如：

> 邀请好友注册并完成首次充值后，您可得 $20，好友可得 $10 API 额度。无上限、不过期，任意邮箱均可。

当 `inviter_reward_max_count` 大于 `0` 时，以“最多可获得 N 次邀请奖励”替换“无上限”，避免展示与配置冲突的信息。“不过期，任意邮箱均可”保持固定。

## 前端结构

在 `web/default/src/features/invitations/` 内新增邀请奖励摘要组件，并将其放在 `SectionPageLayout.Content` 的首项，使其在视觉上紧邻页面标题。组件只负责根据 `InvitationSummary` 选择文案并格式化金额，不修改共享的 `SectionPageLayout`。

邀请页复用现有查询结果，不新增请求或后端字段：

1. `Invitations` 从现有查询响应取得 `summary`。
2. 将 `summary` 传给奖励摘要组件。
3. 组件使用现有 `formatInvitationUSD` 格式化金额。
4. 金额相同与不同、无限与有限上限分别使用可翻译的文案键。

## 加载与失败状态

- 首次加载时显示与单行说明接近的骨架，避免布局跳动。
- 邀请摘要接口失败且没有真实配置时不显示金额或“无上限”，也不回退到硬编码默认值。
- 分页切换时保留已加载摘要，不因记录列表占位状态隐藏说明。

## 国际化

所有文案通过 `t()` 渲染，并同步到 `en`、`zh`、`fr`、`ru`、`ja`、`vi`、`es`、`pt` 八个语言文件。每种翻译必须保留金额和次数插值占位符，不使用英文占位翻译。

## 测试

实施遵循测试驱动：

1. 相同金额且无限次时显示“双方各得”和“无上限”。
2. 不同金额时分别显示邀请者和受邀者金额。
3. 有限次数时显示配置次数，不显示“无上限”。
4. 加载或失败且无摘要时不展示伪造金额与上限。
5. 八种语言均包含新文案，且保留全部插值占位符。
6. 运行邀请页前端测试、类型检查、lint、i18n 同步检查和构建。

## 非目标

- 不修改邀请奖励发放、首次充值触发或额度转余额逻辑。
- 不修改奖励金额和邀请次数上限的后台配置入口。
- 不新增后端接口、数据库字段、依赖或共享布局能力。
- 不修改 `web/classic` 或 `website`。

## 完成标准

- 标题下方说明准确反映当前双方奖励金额和邀请次数上限。
- 配置变化后刷新页面即可看到新文案，无硬编码 `$20` 或无限次数。
- 加载和错误状态不展示错误的业务事实。
- 八种语言与目标测试、类型检查、lint 和构建均通过。
