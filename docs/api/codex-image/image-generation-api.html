图像生成 API（OpenAI 兼容）

通过标准 OpenAI 图像接口（/v1/images/generations 与 /v1/images/edits）生成或编辑图片。请求/响应与 OpenAI Images API 兼容，标准 OpenAI SDK 可直接迁移；结果统一以 b64_json 形式返回。模型名以你的服务实际开放为准（如 gpt-image-2 或 openai/gpt-image-2）。

1. 概览

项值
可用模型以你的服务实际开放的模型名为准（如 gpt-image-2 / openai/gpt-image-2）
调用模式同步：一次请求直接返回图片，客户端无需轮询（慢图由网关内部处理，对客户端透明）。可选 stream:true 增量返回，见第 6 节
能力文生图（generations）、图生图 / 局部编辑（edits，支持可选 mask）
请求格式与官方 OpenAI Images API 一致：generations 为 JSON，edits 为 multipart/form-data
结果交付图片统一以 base64 返回（data[].b64_json，url 为空），附 revised_prompt（模型改写后的提示词）
计费按量计费（按张或按 token，取决于服务/渠道配置）；具体单价见控制台 / 套餐

下文中的 {BASE} 指服务地址，例如 https://your-api-host。

2. 认证

所有请求需在请求头携带访问令牌：

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

3. 接口一览

方法 & 路径说明Content-Type鉴权
POST/v1/images/generations文生图application/jsonBearer
POST/v1/images/edits图生图 / 编辑（含可选 mask）multipart/form-dataBearer

要让某个渠道提供图像能力，需在该渠道的「模型」列表中加入对应的图像模型名（如 gpt-image-2 / openai/gpt-image-2）。上游的承载/计费细节对客户端透明。

4. 文生图

POST{BASE}/v1/images/generations

请求参数（JSON Body）

字段类型必填说明
modelstring必填图像模型名（如 gpt-image-2 / openai/gpt-image-2，以服务开放为准）
promptstring必填图像描述
streambool可选true 时以 SSE 增量返回（最终事件携带图片）；不传为同步返回。见第 6 节
sizestring可选输出尺寸，如 1024x1024
qualitystring可选画质档位：low / medium / high（不传由上游默认）。档位越高 token / 费用越多，见第 6 节
backgroundstring可选背景处理，如 transparent / opaque（透传上游）
output_formatstring可选图片格式，如 png / jpeg / webp（透传上游）
output_compressionint可选压缩级别（透传上游）
moderationstring可选内容审核档位（透传上游）
response_formatstring可选结果恒以 b64_json 返回；url 不提供（见第 8 节）
nint可选返回张数，按张计费；个别模型 / 渠道恒返 1 张（见第 8 节）

请求示例

curl {BASE}/v1/images/generations \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "a photorealistic red fox in a snowy forest at golden hour, highly detailed",
    "size": "1024x1024",
    "quality": "high"
  }'

响应

{
  "created": 1781096207,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAA...（base64 编码的 PNG）...RK5CYII=",
      "revised_prompt": "A photorealistic red fox sitting in a snowy forest at golden hour...",
      "url": ""
    }
  ]
}

图片在 data[0].b64_json（base64）。url 字段恒为空字符串——本接口只回 base64，不提供托管 URL。revised_prompt 为模型实际使用的改写提示词。

5. 图生图（编辑）

POST{BASE}/v1/images/edits · multipart/form-data

表单字段

字段类型必填说明
modeltext必填图像模型名（如 gpt-image-2 / openai/gpt-image-2，以服务开放为准）
prompttext必填编辑指令
imagefile必填待编辑图片（PNG / JPEG / WEBP）。多图可用 image[] / image[0]…
maskfile可选蒙版图：透明区域为待重绘区域。提供后做局部重绘（inpainting）
sizetext可选输出尺寸
qualitytext可选画质档位，同第 4 节

若提供了 mask 但服务端无法读取它（损坏 / 超限），请求会直接报错，不会退化成整图编辑——避免「编辑结果错误却照常计费」。

请求示例

curl {BASE}/v1/images/edits \
  -H "Authorization: Bearer sk-xxxx" \
  -F model=gpt-image-2 \
  -F prompt="add a small yellow star in the center" \
  -F image=@input.png \
  -F mask=@mask.png

响应

{
  "created": 1781096428,
  "data": [
    {
      "b64_json": "iVBORw0KGgoAAAANSUhEUgAA...（编辑后的 PNG, base64）...",
      "revised_prompt": "Edit the provided image: add a small yellow five-pointed star exactly in the center...",
      "url": ""
    }
  ]
}

edits 的输入图本身也会计入用量，因此 edits 通常比纯文生图更贵。

6. 流式响应（可选）

在 generations / edits 请求中加 "stream": true，响应即为 text/event-stream（SSE）。连接建立后服务端会在生成期间发送心跳保活，生成完成时推送终态事件、最后以 data: [DONE] 结束。适合慢图（避免客户端 / 代理在长生成期间超时断开）。

事件说明
image_generation.completed文生图完成；data 字段携带图片（b64_json），多图各一条
image_edit.completed图生图（edits）完成，同上
[DONE]流结束标记

不传 stream 即为同步：一次请求阻塞到出图，无需关心上述事件。流式是否可用以服务 / 渠道为准。

7. 计费与质量

计费按你的服务 / 渠道配置（按张或按 token）；最终单价见控制台 / 套餐。成本与耗时随 quality / size 上升：

quality相对成本典型耗时（1024×1024）
standard / 默认1×（基准）~17–30s
high显著更高（可达数倍）~60–70s
edits（含输入图）输入图额外计入~45s

定价注意：high 质量图成本可达默认质量的数倍。若客户端可自由指定 quality:high，请据此设定合理价格 / 倍率，避免成本失控。

部分上游账号档位可能将 high 实际降级为中等画质——网关如实透传 quality 并按上游真实用量计费。

8. 限制与约定

约定说明
n 多张多数模型按 n 返回对应张数并按张计费；个别模型 / 渠道恒返 1 张（不报错）。如需确定行为，建议 n=1 多次调用
仅 b64_json结果统一以 base64 返回，不提供托管 URL；response_format:"url" 不生效（视渠道为忽略或拒绝）
模型名需使用服务实际开放的图像模型名（如 gpt-image-2 / openai/gpt-image-2）；其他模型名会被拒绝
白标上游错误不透传给客户端：失败时仅返回通用错误消息，上游品牌 / host / 内部模型名不外泄
高质量延迟high 出图可达 ~60–70s，请确保客户端 / 代理 / 网关超时足够（建议 ≥150s，或用 stream:true 保活）

9. 错误

请求 / 提交阶段错误走标准错误体：

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
response_format:"url"不生效：结果恒为 b64_json（视渠道忽略该参数或返回 HTTP 400）
未开放的模型名需使用服务开放的图像模型名，否则被拒
edits 缺少 imagemultipart 中未提供图片文件
上游失败（4xx/5xx）返回通用错误消息（白标），详情记录在服务端日志
额度不足请求被拒

10. 完整示例（文生图 → 落地为图片文件）

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 文生图，取出 b64 并解码为 PNG
curl -sS "$BASE/v1/images/generations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "a cute corgi puppy sitting on grass",
    "size": "1024x1024"
  }' \
| jq -r '.data[0].b64_json' \
| base64 -d > output.png

echo "saved -> output.png"
