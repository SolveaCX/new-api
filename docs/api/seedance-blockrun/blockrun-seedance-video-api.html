异步视频生成 API

Seedance 视频生成 API

调用视频生成接口生成视频（文生 / 图生 / 首尾帧 / 多图参考 / 角色一致）。请求体与主流 Seedance 视频 API 一致，采用「提交任务 → 轮询状态 → 取回视频」的异步模型；视频经本服务代理返回，可直接嵌入页面。部分高级参数与输入模式不可用（参考视频 / 参考音频、画质超分、联网增强、角色登记）——以下逐项标注。

1. 能力总览（支持 / 不支持）

本服务在标准 Seedance content[] 协议基础上，仅开放下列能力。「不支持」项在提交时即报错（提前失败，不扣费）；「忽略」项可正常提交但参数不生效。

能力 / 参数状态说明
文生视频（text）✅ 支持纯提示词生成
图生视频（单图 image_url，无 role）✅ 支持单张种子图，图生视频模式
角色 / 真人一致（real_face_asset_id）✅ 支持透传已有 ta_ 资产；仅 seedance-2.0 / seedance-2.0-fast；与图片输入互斥
分辨率（resolution）✅ 支持360p / 480p / 720p / 1080p / 4K（默认 720p）
时长（duration）✅ 支持秒；默认值 / 上限由所选模型决定（不传用模型默认）
同步音频（generate_audio）✅ 支持是否生成同步音频
画面比例（aspect_ratio）✅ 支持客户端字段名 ratio，值 adaptive / 16:9 / 9:16 / 1:1 / 4:3 / 3:4 / 21:9 / 9:21
随机种子（seed）✅ 支持固定种子可复现结果
水印（watermark）✅ 支持是否添加水印
返回尾帧（return_last_frame）✅ 支持返回尾帧静帧
首帧 / 尾帧插值（first_frame / last_frame）✅ 支持各最多 1 张；last_frame 需配 first_frame；不能与参考图混用
多图参考 / omni（2–9 张 reference_image）✅ 支持2–9 张参考图；单张需显式 role: reference_image；仅 seedance-2.0 / seedance-2.0-fast
参考视频（video_url）❌ 不支持提交报错
参考音频（audio_url）❌ 不支持提交报错
角色 / 真人登记（生成 ta_ 资产）❌ 暂不提供本服务不含登记接口；ta_ 资产需线下预先获取后透传
画质超分（super_resolution_config）❌ 不支持不生效，亦无独立超分子任务
联网增强（web_search）❌ 不支持—
camera_fixed / frames / callback_url / input_type⚠️ 忽略为兼容而接受，但传入不生效
token 用量（usage）❌ 不返回本服务按次计费，结果中不含 usage 字段

2. 概览

项值
可用模型seedance-2.0、seedance-2.0-fast、seedance-1.5-pro（以你的服务实际开放的模型名为准）
调用模式异步：创建任务 → 轮询 → 取视频
能力文生视频、图生视频（单图）、首尾帧插值、多图参考（omni，2.0 系）、角色 / 真人一致（透传 ta_ 资产）
请求格式多模态 content[] 数组（与主流 Seedance API 一致，便于无缝迁移）
结果交付视频经本服务代理地址返回，可直接下载或嵌入
不含参考视频 / 参考音频 / 画质超分 / 联网增强 / 角色登记接口（详见第 1、6 节）

下文中的 {BASE} 指服务地址，例如 https://your-api-host。

3. 认证

除「获取视频」接口外，所有请求需在请求头携带访问令牌：

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

例外：「获取视频」接口 GET {BASE}/v1/videos/{task_id}/content 为匿名接口，无需令牌 —— 不可猜的 task_id 本身即访问凭证，可直接用于 <video src> 嵌入。

4. 接口一览

方法 & 路径说明鉴权
POST/v1/videos创建视频生成任务Bearer
GET/v1/videos/{task_id}查询任务状态 / 结果Bearer
GET/v1/videos/{task_id}/content取回视频二进制（代理下载）匿名

另有一组等价接口 POST /v1/video/generations 与 GET /v1/video/generations/{task_id}，创建行为完全相同，仅查询响应为服务私有格式。推荐使用上方 /v1/videos 这套（标准结构）。

5. 创建视频任务

POST{BASE}/v1/videos

请求参数（JSON Body）

字段类型必填状态说明
modelstring必填✅seedance-2.0 / seedance-2.0-fast / seedance-1.5-pro（以服务开放的名称为准）
contentarray必填✅多模态输入数组，见下「content 元素」。至少含一条文本，或一张图片
resolutionstring可选✅360p / 480p / 720p / 1080p / 4K（默认 720p，大小写不敏感）。其他值提交报错
durationint可选✅时长（秒）；默认值 / 上限由所选模型决定；不传用模型默认
generate_audiobool可选✅是否生成同步音频
real_face_asset_idstring可选✅顶层字段。角色 / 真人一致资产（ta_ 前缀）。仅 seedance-2.0 / seedance-2.0-fast；与图片输入互斥；资产需线下预先持有（本服务不提供登记）
ratiostring可选✅画面比例（映射到上游 aspect_ratio）。值 adaptive / 16:9 / 9:16 / 1:1 / 4:3 / 3:4 / 21:9 / 9:21
seedint可选✅随机种子；固定后可复现结果
watermarkbool可选✅是否添加水印
return_last_framebool可选✅是否返回尾帧静帧
camera_fixed / frames / callback_url / input_type / web_search / super_resolution_config—可选❌/⚠️不支持 / 忽略，详见第 6 节

content 元素

type状态附带字段 / role说明
text✅ 支持text（string）文本提示词（可有多条，按顺序拼接）
image_url✅ 支持image_url.url + role按 role 决定模式：无 role 单张 = 图生视频；first_frame(+last_frame) = 首尾帧插值；reference_image = 多图参考（2–9 张，仅 2.0 系）。约束见第 6 节
video_url❌ 不支持—含此项即提交报错：「video input is not supported」
audio_url❌ 不支持—含此项即提交报错：「audio input is not supported」

输入模式（4 种，图像模式互斥）：① 文生（仅 text）；② 图生视频（单张 image_url，无 role）；③ 首尾帧插值（first_frame 必填、last_frame 可选）；④ 多图参考 / omni（2–9 张 role: reference_image，仅 seedance-2.0 / seedance-2.0-fast）。②③④ 三种图像模式不能混用。不支持参考视频 / 参考音频；传入会在提交阶段直接报错（提前失败，不扣费）。

请求示例（文生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感，镜头缓慢推近" }
    ],
    "resolution": "1080p",
    "duration": 5
  }'

请求示例（单图图生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "镜头缓缓推进，光影流动" },
      { "type": "image_url", "image_url": { "url": "https://example.com/ref.jpg" }, "role": "reference_image" }
    ],
    "resolution": "720p",
    "duration": 5
  }'

请求示例（首尾帧插值）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "花苞缓缓绽放" },
      { "type": "image_url", "image_url": { "url": "https://example.com/first.jpg" }, "role": "first_frame" },
      { "type": "image_url", "image_url": { "url": "https://example.com/last.jpg" },  "role": "last_frame" }
    ],
    "resolution": "720p",
    "duration": 5
  }'

请求示例（多图参考 / omni · 仅 2.0 系）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "让 image 1 的角色穿上 image 2 的服装" },
      { "type": "image_url", "image_url": { "url": "https://example.com/char.jpg" },  "role": "reference_image" },
      { "type": "image_url", "image_url": { "url": "https://example.com/cloth.jpg" }, "role": "reference_image" }
    ],
    "resolution": "1080p",
    "duration": 5
  }'

②③④ 三种图像输入模式互斥：单张无 role 图（图生视频）、首尾帧（first_frame / last_frame）、多图参考（reference_image）不能混用。多图参考及「单张显式 reference_image」仅 seedance-2.0 / seedance-2.0-fast 支持。

请求示例（角色 / 真人一致 · 透传 ta_ 资产）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "real_face_asset_id": "ta_xxxxxxxx",
    "content": [
      { "type": "text", "text": "她在柔和的影棚灯光下微笑挥手" }
    ],
    "resolution": "1080p",
    "duration": 5
  }'

real_face_asset_id 是顶层字段（与 model / content 同级），不放在 content[] 内。仅 seedance-2.0 / seedance-2.0-fast 接受；不能与图片输入同时使用。

响应

{
  "id": "task_3f9a...e2",
  "task_id": "task_3f9a...e2",
  "object": "video",
  "model": "seedance-2.0",
  "status": "queued",
  "progress": 0,
  "created_at": 1780306574
}

id / task_id 是公开任务 ID（task_ 前缀 + 随机串），后续轮询和取视频都用它。

6. 不可用参数与功能（重点）

❌ 不支持 —— 提交阶段直接报错（提前失败，不扣费）

参数 / 功能报错信息替代方案
参考视频 content[].video_urlvideo input is not supported by this channel无（本服务不做视频到视频）
参考音频 content[].audio_urlaudio input is not supported by this channel无
参考图超过 9 张at most 9 reference images are supported≤ 9 张
first_frame / last_frame 各超过 1 张at most one first_frame and one last_frame image are supported各保留 1 张
只给 last_frame 不给 first_framelast_frame requires a first_frame image as the starting frame补一张 first_frame
首尾帧与参考图混用first_frame/last_frame cannot be combined with reference images只用一种图像输入模式
多图 / 显式 reference_image 用于 seedance-1.5-proreference images (omni) are only supported on seedance-2.0 / seedance-2.0-fast换 2.0 / 2.0-fast，或用单张无 role 图
不支持的 resolutionunsupported resolution ...用 360p/480p/720p/1080p/4K
real_face_asset_id 用于 seedance-1.5-proreal_face_asset_id is only supported on seedance-2.0 / seedance-2.0-fast换 2.0 / 2.0-fast

real_face_asset_id 格式错误real_face_asset_id must start with 'ta_'使用合法 ta_ 资产

❌ 暂不提供 —— 角色 / 真人登记

本服务不包含「登记」类接口（即生成 ta_ 资产的发起 / 状态 / 定稿流程）。如需角色或真人一致能力，请线下预先获取 ta_ 资产，再在创建请求里通过 real_face_asset_id 透传。该能力后续版本可能补充。

⚠️ 忽略 —— 可提交但不生效

参数行为
camera_fixed / frames / callback_url / input_type为兼容标准 Seedance 协议而接受，传入无效果
web_search / super_resolution_config不生效（无联网增强、无画质超分子任务）

❌ 不返回 —— token 用量

本服务按调用次数计费，查询结果中不含 usage 字段。请勿依赖 usage.total_tokens 做计量。

7. 查询任务状态

GET{BASE}/v1/videos/{task_id}

curl {BASE}/v1/videos/task_3f9a...e2 \
  -H "Authorization: Bearer sk-xxxx"

排队 / 进行中

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "in_progress",
  "progress": 30,
  "model": "seedance-2.0",
  "created_at": 1780306574
}

完成

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "completed",
  "progress": 100,
  "model": "seedance-2.0",
  "metadata": {
    "url": "{BASE}/v1/videos/task_3f9a...e2/content"
  },
  "created_at": 1780306574,
  "completed_at": 1780306750
}

完成后，视频地址在 metadata.url —— 它是本服务的代理地址，直接 GET 即可拿到视频（见第 8 节）。注：本服务不返回 usage。

失败

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "failed",
  "error": { "message": "..." }
}

任务失败不计费（已预扣的额度自动退还）。错误信息已做脱敏处理。

8. 获取视频

GET{BASE}/v1/videos/{task_id}/content · 匿名

任务成功后，直接 GET 该地址即可流式返回 video/mp4 二进制。无需鉴权，可直接嵌入页面：

# 下载到本地
curl -L "{BASE}/v1/videos/task_3f9a...e2/content" -o output.mp4

# 或在 HTML 中直接嵌入
<video src="{BASE}/v1/videos/task_3f9a...e2/content" controls></video>

仅当任务状态为 completed 时可取；未完成会返回错误。该接口按客户端 IP 限流。

9. 状态值

status含义
queued排队中
in_progress生成中（progress 0–100）
completed已完成，视频在 metadata.url
failed失败，详情在 error.message；已退款

10. 错误

任务级失败体现在查询响应的 error 字段（见第 7 节）。请求 / 提交阶段的错误走标准错误体：

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
content 为空提交被拒：需至少包含一条文本，或一张图片
不支持的输入传入参考视频 / 参考音频，或违反输入模式约束（首尾帧 / 多图参考的张数、组合、模型限制）—— 见第 6 节，提交阶段即报错
模型不可用传入了服务未开放的 model
额度不足任务不会创建
服务暂时不可用通常可稍后重试

11. 完整示例（创建 → 轮询 → 下载）

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 1) 创建任务
RESP=$(curl -sS "$BASE/v1/videos" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感" }
    ],
    "resolution": "1080p",
    "duration": 5
  }')
TASK_ID=$(echo "$RESP" | jq -r '.id')
echo "task: $TASK_ID"

# 2) 轮询直到完成或失败
while true; do
  Q=$(curl -sS "$BASE/v1/videos/$TASK_ID" \
        -H "Authorization: Bearer $TOKEN")
  STATUS=$(echo "$Q" | jq -r '.status')
  echo "status: $STATUS"
  [ "$STATUS" = "completed" ] && break
  if [ "$STATUS" = "failed" ]; then
    echo "failed: $(echo "$Q" | jq -r '.error.message')"; exit 1
  fi
  sleep 5
done

# 3) 取回视频（匿名，直接用 metadata.url）
URL=$(echo "$Q" | jq -r '.metadata.url')
curl -L "$URL" -o output.mp4
echo "saved -> output.mp4"
