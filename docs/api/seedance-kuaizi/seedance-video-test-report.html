反例 1 — 不支持的分辨率 PASS

POST/v1/videos

请求

{
  "model": "Seedance2.0-fast",
  "content": [{"type":"text","text":"测试"}],
  "resolution": "2K"
}

响应 · HTTP 400

{
  "code": "invalid_request",
  "message": "unsupported resolution \"2K\"; supported: 480p / 720p / 1080p (higher resolutions require super_resolution_config)",

}

✅ 提交阶段即拒绝，未发往上游。高于 1080p 需走 super_resolution_config。

5. 观察与备注

usage 数值：三条 completion_tokens = total_tokens = 108900（均 5s / 720p）。上游按"分辨率×时长"给固定 token 计量，新-api 仅原样透传。用例 C 的超分（→1080p）未体现在 usage 上，属上游计量口径，非网关代码问题。

计费：通用格式 quota 字段为 337500（单任务扣费额度），说明定价配置生效、预扣 / 结算正常。

安全提示：视频链接 /v1/videos/{task_id}/content 为匿名访问（task_id 即访问凭证），可直接嵌入 <video>；分享即等于授予下载权限。本报告未包含任何 API Key / 令牌。
