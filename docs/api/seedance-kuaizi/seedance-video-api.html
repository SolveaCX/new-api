异步视频生成 API

Seedance 视频生成 API

调用视频生成接口生成视频（文生 / 图生 / 多模态参考）。请求体与主流视频生成 API 一致，采用「提交任务 → 轮询状态 → 取回视频」的异步模型；视频经本服务代理返回，可直接嵌入页面。

1. 概览

项值
可用模型seedance-2.0、seedance-2.0-fast（以你的服务实际开放的模型名为准）
调用模式异步：创建任务 → 轮询 → 取视频
能力文生视频、图生视频（首帧 / 尾帧 / 参考图）、多模态参考（图片 + 视频 + 音频组合）、可选画质超分
请求格式多模态 content[] 数组（与主流视频生成 API 一致，便于无缝迁移）
结果交付视频经本服务代理地址返回，可直接下载或嵌入；附带 token 用量 usage

下文中的 {BASE} 指服务地址，例如 https://your-api-host。

2. 认证

除「获取视频」接口外，所有请求需在请求头携带访问令牌：

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

例外：「获取视频」接口 GET {BASE}/v1/videos/{task_id}/content 为匿名接口，无需令牌 —— 不可猜的 task_id 本身即访问凭证，可直接用于 <video src> 嵌入。

3. 接口一览

方法 & 路径说明鉴权
POST/v1/videos创建视频生成任务Bearer
GET/v1/videos/{task_id}查询任务状态 / 结果（含 usage）Bearer
GET/v1/videos/{task_id}/content取回视频二进制（代理下载）匿名

另有一组等价接口 POST /v1/video/generations 与 GET /v1/video/generations/{task_id}，创建行为完全相同，仅查询响应为服务私有格式。推荐使用上方 /v1/videos 这套（标准结构、原生携带 usage）。

4. 创建视频任务

POST{BASE}/v1/videos

请求参数（JSON Body）

字段类型必填说明
modelstring必填模型名，如 seedance-2.0 / seedance-2.0-fast（以你的服务开放的名称为准）
contentarray必填多模态输入数组，见下方「content 元素」。至少包含一个文本项，或一张图片 / 一段视频
resolutionstring可选输出分辨率：480p / 720p / 1080p
ratiostring可选画面比例：16:9 / 4:3 / 1:1 / 3:4 / 9:16 / 21:9 / adaptive
durationint可选时长（秒），范围 4–15；传 -1 由模型智能决定
seedint可选随机种子；不传则随机
watermarkbool可选是否添加水印（默认 false）
generate_audiobool可选是否生成同步音频

content 元素

type附带字段role 可选值说明
texttext（string）—文本提示词（可有多条，按顺序拼接）
image_urlimage_url.urlfirst_frame / last_frame / reference_image输入图片，最多 9 张
video_urlvideo_url.urlreference_video参考视频，最多 3 段
audio_urlaudio_url.urlreference_audio参考音频，最多 3 段

输入模式由图片 role 自动推断：含 first_frame / last_frame 时为「首尾帧」模式，否则为「参考图」模式。

进阶参数（可选）

字段类型说明
input_typestring显式指定输入模式：reference / first_last_frame（默认自动推断）
web_searchbool启用联网搜索增强
super_resolution_configobject画质超分配置，非空即启用。任务成功后自动进行画质增强，对外仍只暴露一个任务 ID

super_resolution_config 字段

字段类型说明
resolutionstring目标分辨率：720p / 1080p / 2k / 4k（须高于原始分辨率）。与 resolution_limit 二选一
resolution_limitint自定义短边像素 [64, 2160]。与 resolution 二选一
scenestring场景：aigc / short_series / ugc / old_film
tool_versionstring版本：standard（默认）/ professional
fpsint输出帧率 [1, 120]；高于源帧率时触发智能插帧

注意：camera_fixed、frames、callback_url、return_last_frame 等参数为兼容而接受，但当前不会生效（传入会被忽略）。

请求示例（文生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感，镜头缓慢推近" }
    ],
    "resolution": "1080p",
    "ratio": "16:9",
    "duration": 5
  }'

请求示例（首尾帧图生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "镜头缓缓推进" },
      { "type": "image_url", "image_url": { "url": "https://example.com/first.jpg" }, "role": "first_frame" },
      { "type": "image_url", "image_url": { "url": "https://example.com/last.jpg" },  "role": "last_frame" }
    ],
    "resolution": "720p",
    "ratio": "16:9",
    "duration": 5
  }'

请求示例（多模态参考 + 画质超分）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "女孩对着镜头微笑，背景是海边日落" },
      { "type": "image_url", "image_url": { "url": "https://example.com/ref.jpg" }, "role": "reference_image" }
    ],
    "resolution": "1080p",
    "ratio": "16:9",
    "duration": 5,
    "watermark": false,
    "super_resolution_config": { "resolution": "4k", "scene": "aigc", "tool_version": "professional", "fps": 60 }
  }'

响应

{
  "id": "task_3f9a...e2",
  "task_id": "task_3f9a...e2",
  "object": "video",
  "model": "seedance-2.0",
  "status": "queued",
  "progress": 0,
  "created_at": 1780306574
}

id / task_id 是公开任务 ID（task_ 前缀 + 32 位随机串），后续轮询和取视频都用它。

5. 查询任务状态

GET{BASE}/v1/videos/{task_id}

curl {BASE}/v1/videos/task_3f9a...e2 \
  -H "Authorization: Bearer sk-xxxx"

排队 / 进行中

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "in_progress",
  "progress": 50,
  "model": "seedance-2.0",
  "created_at": 1780306574
}

完成

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "completed",
  "progress": 100,
  "model": "seedance-2.0",
  "usage": {
    "completion_tokens": 120,
    "total_tokens": 120
  },
  "metadata": {
    "url": "{BASE}/v1/videos/task_3f9a...e2/content"
  },
  "created_at": 1780306574,
  "completed_at": 1780306750
}

完成后，视频地址在 metadata.url —— 它是本服务的代理地址，直接 GET 即可拿到视频（见第 6 节）。usage 为本次生成消耗的 token 用量。

失败

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "failed",
  "error": { "message": "...", "code": "..." }
}

任务失败不计费（已预扣的额度自动退还）。

6. 获取视频

GET{BASE}/v1/videos/{task_id}/content · 匿名

任务成功后，直接 GET 该地址即可流式返回 video/mp4 二进制。无需鉴权，可直接嵌入页面：

# 下载到本地
curl -L "{BASE}/v1/videos/task_3f9a...e2/content" -o output.mp4

# 或在 HTML 中直接嵌入
<video src="{BASE}/v1/videos/task_3f9a...e2/content" controls></video>

仅当任务状态为 completed 时可取；未完成会返回错误。该接口按客户端 IP 限流。

7. 状态值

status含义
queued排队中
in_progress生成中（progress 0–100）
completed已完成，视频在 metadata.url，token 用量在 usage
failed失败，详情在 error.message；已退款

8. 错误

任务级失败体现在查询响应的 error 字段（见第 5 节）。请求 / 提交阶段的错误走标准错误体：

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
content 为空提交被拒：需至少包含一条文本，或一张图片 / 一段视频
模型不可用传入了服务未开放的 model
额度不足任务不会创建
服务暂时不可用通常可稍后重试

9. 完整示例（创建 → 轮询 → 下载）

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 1) 创建任务
RESP=$(curl -sS "$BASE/v1/videos" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感" }
    ],
    "resolution": "1080p",
    "ratio": "16:9",
    "duration": 5
  }')
TASK_ID=$(echo "$RESP" | jq -r '.id')
echo "task: $TASK_ID"

# 2) 轮询直到完成或失败
while true; do
  Q=$(curl -sS "$BASE/v1/videos/$TASK_ID" \
        -H "Authorization: Bearer $TOKEN")
  STATUS=$(echo "$Q" | jq -r '.status')
  echo "status: $STATUS"
  [ "$STATUS" = "completed" ] && break
  if [ "$STATUS" = "failed" ]; then
    echo "failed: $(echo "$Q" | jq -r '.error.message')"; exit 1
  fi
  sleep 5
done

# 3) 取回视频（匿名，直接用 metadata.url）
URL=$(echo "$Q" | jq -r '.metadata.url')
TOKENS=$(echo "$Q" | jq -r '.usage.total_tokens')
echo "tokens: $TOKENS"
curl -L "$URL" -o output.mp4
echo "saved -> output.mp4"
