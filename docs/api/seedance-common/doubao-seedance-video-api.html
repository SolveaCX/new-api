异步视频生成 API

Doubao Seedance 视频生成 API

调用 seedance 视频生成接口生成视频（文生 / 图生 / 多模态参考）。请求体采用官方 seedance content[] 格式，遵循「提交任务 → 轮询状态 → 取回视频」的异步模型；任务完成后返回带签名时效的视频直链，可直接下载或嵌入页面，并附带 token 用量 usage。

1. 概览

项值
可用模型doubao-seedance-2-0-260128、doubao-seedance-2-0-fast-260128 等（以你的服务实际开放的模型名为准）
调用模式异步：创建任务 → 轮询 → 取视频
能力文生视频、图生视频（首帧 / 尾帧 / 参考图）、多模态参考（图片 + 视频 + 音频组合）、可选同步音频
请求格式官方 seedance 多模态 content[] 数组（与官方 seedance API 完全一致，便于无缝迁移）
结果交付任务完成后视频地址在 metadata.url —— 为带签名时效的对象存储直链（默认约 24h），直接 GET 即可下载或嵌入；附带 token 用量 usage

下文中的 {BASE} 指服务地址，例如 https://your-api-host。

2. 认证

所有请求需在请求头携带访问令牌：

Authorization: Bearer sk-xxxxxxxxxxxxxxxx

3. 接口一览

方法 & 路径说明鉴权
POST/v1/videos创建视频生成任务Bearer
GET/v1/videos/{task_id}查询任务状态 / 结果（含 usage 与 metadata.url）Bearer

另有一组等价接口 POST /v1/video/generations 与 GET /v1/video/generations/{task_id}，创建行为完全相同，仅查询响应为服务私有格式。推荐使用上方 /v1/videos 这套（标准结构、原生携带 usage）。

4. 创建视频任务

POST{BASE}/v1/videos

请求参数（JSON Body）

字段类型必填说明
modelstring必填模型名，如 doubao-seedance-2-0-260128 / doubao-seedance-2-0-fast-260128（以你的服务开放的名称为准）
contentarray必填多模态输入数组，见下方「content 元素」。至少包含一个文本项，或一张图片 / 一段视频
resolutionstring可选输出分辨率：480p / 720p / 1080p
ratiostring可选画面比例：16:9 / 4:3 / 1:1 / 3:4 / 9:16 / 21:9 / adaptive
durationint可选时长（秒），如 5 / 10；传 -1 由模型智能决定
framesint可选总帧数（与 duration 配合，按模型规则取其一）
seedint可选随机种子；不传则随机
watermarkbool可选是否添加水印（默认 false）
generate_audiobool可选是否生成同步音频
camera_fixedbool可选是否固定镜头（关闭运镜）
return_last_framebool可选是否额外返回尾帧图
callback_urlstring可选任务完成回调地址（设置后上游会主动回调）

content 元素

type附带字段role 可选值说明
texttext（string）—文本提示词（可有多条，按顺序拼接）
image_urlimage_url.urlfirst_frame / last_frame / reference_image输入图片，最多 9 张
video_urlvideo_url.urlreference_video参考视频，最多 3 段
audio_urlaudio_url.urlreference_audio参考音频，最多 3 段

输入模式由图片 role 自动推断：含 first_frame / last_frame 时为「首尾帧」模式，否则为「参考图」模式。

进阶参数（可选）

字段类型说明
toolsarray工具列表，如 [{"type":"web_search"}] 启用联网搜索增强
service_tierstring服务档位（按上游开放的档位取值）
draftbool草稿 / 快速预览模式
execution_expires_afterint任务执行超时时间（秒），超时未完成则失败

进阶参数为本渠道扩展，纯官方 seedance 调用者可全部省略。未提供的可选字段一律不下发，由上游使用默认值。

请求示例（文生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感，镜头缓慢推近" }
    ],
    "resolution": "1080p",
    "ratio": "16:9",
    "duration": 5
  }'

请求示例（首尾帧图生视频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "content": [
      { "type": "text", "text": "镜头缓缓推进" },
      { "type": "image_url", "image_url": { "url": "https://example.com/first.jpg" }, "role": "first_frame" },
      { "type": "image_url", "image_url": { "url": "https://example.com/last.jpg" },  "role": "last_frame" }
    ],
    "resolution": "720p",
    "ratio": "16:9",
    "duration": 5
  }'

请求示例（多模态参考：图 + 视频 + 音频）

curl {BASE}/v1/videos \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "content": [
      { "type": "text",      "text": "第一人称果茶宣传广告，参考视频的运镜、音频作背景乐" },
      { "type": "image_url", "image_url": { "url": "https://example.com/pic1.jpg" }, "role": "reference_image" },
      { "type": "image_url", "image_url": { "url": "https://example.com/pic2.jpg" }, "role": "reference_image" },
      { "type": "video_url", "video_url": { "url": "https://example.com/ref.mp4" },  "role": "reference_video" },
      { "type": "audio_url", "audio_url": { "url": "https://example.com/bgm.mp3" },  "role": "reference_audio" }
    ],
    "ratio": "16:9",
    "duration": 5,
    "watermark": false,
    "generate_audio": true
  }'

响应

{
  "id": "task_3f9a...e2",
  "task_id": "task_3f9a...e2",
  "object": "video",
  "model": "doubao-seedance-2-0-260128",
  "status": "queued",
  "progress": 0,
  "created_at": 1780306574
}

id / task_id 是公开任务 ID（task_ 前缀 + 32 位随机串），后续轮询和取视频都用它。

5. 查询任务状态

GET{BASE}/v1/videos/{task_id}

curl {BASE}/v1/videos/task_3f9a...e2 \
  -H "Authorization: Bearer sk-xxxx"

排队 / 进行中

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "in_progress",
  "progress": 50,
  "model": "doubao-seedance-2-0-260128",
  "created_at": 1780306574
}

完成

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "completed",
  "progress": 100,
  "model": "doubao-seedance-2-0-260128",
  "usage": {
    "completion_tokens": 216900,
    "total_tokens": 216900
  },
  "metadata": {
    "url": "https://...example-storage.../seedance/xxxx.mp4?X-Tos-Expires=86400&X-Tos-Signature=..."
  },
  "created_at": 1780306574,
  "completed_at": 1780306851
}

完成后，视频地址在 metadata.url —— 它是带签名时效的对象存储直链（默认有效期约 24h），直接 GET 即可下载或用于 <video src> 嵌入。usage 为本次生成消耗的 token 用量。

注意时效：该直链带签名过期时间（X-Tos-Expires，默认约 24h，以上游签名为准）。任务完成后结果即冻结，重新 GET /v1/videos/{task_id} 返回的是同一个地址、不会刷新签名。请在有效期内尽快下载并自行落地存储；签名过期后该直链失效且无法通过本接口续签。

失败

{
  "id": "task_3f9a...e2",
  "object": "video",
  "status": "failed",
  "error": { "message": "...", "code": "..." }
}

任务失败不计费（已预扣的额度自动退还）。

6. 获取视频

任务 completed 后，直接对 metadata.url 发起 GET 即可拿到 video/mp4：

# 从查询响应取出直链
URL=$(curl -sS "{BASE}/v1/videos/task_3f9a...e2" \
        -H "Authorization: Bearer sk-xxxx" | jq -r '.metadata.url')

# 下载到本地
curl -L "$URL" -o output.mp4

# 或在 HTML 中直接嵌入（直链含签名时效，过期前请尽快落地存储）
<video src="$URL" controls></video>

仅当任务状态为 completed 时 metadata.url 才有值。直链含签名时效（默认约 24h），过期后无法通过本接口续签，请在有效期内下载并自行落地存储。

7. 状态值

status含义
queued排队中
in_progress生成中（progress 0–100）
completed已完成，视频在 metadata.url，token 用量在 usage
failed失败，详情在 error.message；已退款

8. 错误

任务级失败体现在查询响应的 error 字段（见第 5 节）。请求 / 提交阶段的错误走标准错误体：

{ "error": { "message": "...", "type": "...", "code": "..." } }
典型场景说明
content 为空提交被拒：需至少包含一条文本，或一张图片 / 一段视频
模型不可用传入了服务未开放的 model
额度不足任务不会创建
服务暂时不可用通常可稍后重试

9. 完整示例（创建 → 轮询 → 下载）

#!/usr/bin/env bash
BASE="https://your-api-host"
TOKEN="sk-xxxxxxxxxxxxxxxx"

# 1) 创建任务
RESP=$(curl -sS "$BASE/v1/videos" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2-0-260128",
    "content": [
      { "type": "text", "text": "宇航员在月球漫步，电影感" }
    ],
    "resolution": "1080p",
    "ratio": "16:9",
    "duration": 5
  }')
TASK_ID=$(echo "$RESP" | jq -r '.id')
echo "task: $TASK_ID"

# 2) 轮询直到完成或失败
while true; do
  Q=$(curl -sS "$BASE/v1/videos/$TASK_ID" \
        -H "Authorization: Bearer $TOKEN")
  STATUS=$(echo "$Q" | jq -r '.status')
  echo "status: $STATUS"
  [ "$STATUS" = "completed" ] && break
  if [ "$STATUS" = "failed" ]; then
    echo "failed: $(echo "$Q" | jq -r '.error.message')"; exit 1
  fi
  sleep 5
done

# 3) 取回视频（metadata.url 为带签名时效的直链）
URL=$(echo "$Q" | jq -r '.metadata.url')
TOKENS=$(echo "$Q" | jq -r '.usage.total_tokens')
echo "tokens: $TOKENS"
curl -L "$URL" -o output.mp4
echo "saved -> output.mp4"
